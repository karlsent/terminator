# Terminator — веб-интерфейс для запуска Claude-агентов

Локальный веб-сервер для управления AI-агентами на базе Claude CLI. Запускает агентов, стримит логи в реальном времени, хранит историю запусков, собирает фидбек и накапливает базу опыта.

## Что умеет

- **Запуск агентов** — задачи, тесты, code review, синтез опыта
- **Стриминг логов** — вывод агента появляется в браузере в реальном времени
- **История** — все запуски с фильтрами, оценками (звёзды) и комментариями
- **Просмотр заметок** — агент пишет `.md` файлы, открываются прямо в браузере с рендером Markdown
- **Синтезатор** — агент читает все фидбеки и обновляет базу опыта (`agent_experience.md`)
- **Две темы** — Agent 007 (классика) и T-800 (sci-fi)
- **Деплой на сервер** — загрузка файлов в Kubernetes-поды через встроенный скрипт
- **Прокси** — поддержка подписочного прокси для Claude CLI

## Требования

- Python 3.8+
- [Claude CLI](https://claude.ai/code) — установлен и авторизован (`claude` в PATH)
- nginx — опционально, для доступа по домену `terminator.agent`

## Установка

```bash
git clone git@github.com:karlsent/terminator.git
cd terminator
python3 install.py
```

Установщик интерактивно спросит:
- путь к git-репозиторию проекта
- использовать ли прокси для Claude
- настроить ли nginx и автозапуск

После установки открыть в браузере: **http://localhost:8765** или **http://terminator.agent** (если настроен nginx).

## Структура

```
terminator.py          # Flask-сервер, весь бэкенд и HTML-шаблон
install.py             # Установщик (копирует файлы, настраивает nginx/systemd/LaunchAgent)
agents/                # Скрипты агентов (.sh) и их промпты (.md)
kube/                  # Скрипты для работы с Kubernetes
proxy/                 # Клиент подписочного прокси
static/                # Логотипы тем
```

Данные хранятся в `~/.terminator/`:
```
config.json            # Настройки
logs/                  # Логи запусков агентов
notes/                 # Заметки агентов (.md файлы)
launcher_feedback.json # Фидбеки и оценки
agent_experience.md    # База опыта (обновляется Синтезатором)
```

## Агенты

| Агент | Назначение |
|-------|-----------|
| `task_agent` | Выполнение задач в git-репозитории |
| `test_agent` | Запуск тестов и анализ результатов |
| `write_tests_agent` | Написание тестов для кода |
| `code_review_agent` | Code review изменений |
| `feedback_synth_agent` | Синтез фидбеков в базу опыта |

## Настройки

Все параметры доступны в вкладке **Настройки** веб-интерфейса: git-репозиторий, прокси, пути к директориям, Kubernetes-кластеры, директории поиска файлов и т.д.
