## Шаг 1 — Загрузить задачи из Bitrix24

URL уже подставлен в команды ниже. **Для каждого ID** из списка `{{TASK_ID}}` выполни:

```bash
curl -s --noproxy "*" "BITRIX_REST_PLACEHOLDER/tasks.task.get?taskId=<ID>&select[]=*"
```

**Проверь ответ**: если `result.item` пустой или присутствует `error` — выведи предупреждение и пропусти эту задачу:
```
[FAIL] Задача <ID> не найдена или недоступна через API. Пропускаю.
```
Если все задачи недоступны — останови работу с `[FATAL]`.

Из каждого успешного ответа запомни: `id`, `title`, `description`, `status`, `deadline`, `responsibleId`, `createdBy`, `parentId`, `dependsOn`, `containsRelatedTasks`, `archiveLink`.

**После загрузки всех задач** — проанализируй их совместно: найди общую тему, пересечения в функциональности и зависимости между ними. Это определит стратегию реализации.

### Комментарии

```bash
curl -s --noproxy "*" "BITRIX_REST_PLACEHOLDER/task.commentitem.getlist?TASKID={{TASK_ID}}&order[DATE_CREATE]=ASC"
```

### Подзадачи

```bash
curl -s --noproxy "*" "BITRIX_REST_PLACEHOLDER/tasks.task.list?filter[PARENT_ID]={{TASK_ID}}&select[]=ID&select[]=TITLE&select[]=STATUS&select[]=RESPONSIBLE_ID&select[]=DEADLINE"
```

### Связанные задачи

Из ответа `tasks.task.get` читай:
- `result.item.dependsOn` — массив ID задач от которых зависит эта
- `result.item.containsRelatedTasks` — есть ли связанные задачи (bool)
- `result.item.containsGanttLinks` — есть ли связи в Гантте (bool)

Для каждого ID из `dependsOn` загрузи задачу и комментарии:
```bash
curl -s --noproxy "*" "BITRIX_REST_PLACEHOLDER/tasks.task.get?taskId=<RELATED_ID>&select[]=*&select[]=UF_*"
curl -s --noproxy "*" "BITRIX_REST_PLACEHOLDER/task.commentitem.getlist?TASKID=<RELATED_ID>&order[DATE_CREATE]=ASC"
```

Используй данные связанных задач только как контекст — не выполняй их.

### Вложенные файлы

Из ответа `tasks.task.get` возьми:
- `result.item.ufTaskWebdavFiles` — массив ID вложений (основной способ)
- `result.item.archiveLink` — запасной вариант (архив всех файлов)

### Способ 1 — через ufTaskWebdavFiles (приоритет)

Если `ufTaskWebdavFiles` не пустой — для каждого ID запроси метаданные:
```bash
curl -s --noproxy "*" "BITRIX_REST_PLACEHOLDER/disk.attachedObject.get?id=<FILE_ID>"
```

Из ответа возьми `NAME` и `DOWNLOAD_URL`. Скачай файл:
```bash
mkdir -p "{{ATTACHMENTS_DIR}}/{{TASK_ID}}"
curl -s --noproxy "*" -L "<DOWNLOAD_URL>" \
  -o "{{ATTACHMENTS_DIR}}/{{TASK_ID}}/<NAME>"
```

### Способ 2 — через archiveLink (если ufTaskWebdavFiles пустой)

Если `archiveLink` не пустой:
```bash
mkdir -p "{{ATTACHMENTS_DIR}}"
curl -s --noproxy "*" -L -o "{{ATTACHMENTS_DIR}}/{{TASK_ID}}_files.zip" \
  "<archiveLink>"
unzip -o "{{ATTACHMENTS_DIR}}/{{TASK_ID}}_files.zip" \
  -d "{{ATTACHMENTS_DIR}}/{{TASK_ID}}/"
```

Если оба поля пустые — файлов нет, запиши: `[INFO] Вложений в задаче нет`

### Анализ файлов

После скачивания прочитай каждый файл через Read и проанализируй:
- **Изображения** (jpg, jpeg, png, gif, webp, bmp) — Read, опиши что изображено
- **PDF, DOCX, TXT, CSV** — Read, опиши содержимое
- **Остальные** — укажи только имя и тип, не читай

Логируй в `{{LOGS_DIR}}/{{TASK_ID}}.log`:
- Успех: `[OK] Скачано файлов: <N>`
- Ошибка: `[FAIL] Не удалось скачать файл <NAME> — <причина>`
